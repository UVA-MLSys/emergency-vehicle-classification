This project looks to expand on the Point Pillars methodology of autonomous vehicles with respect to emergency vehicles

Point Pillars

At a glance, Point Pillars relies on data from LiDaR (Light Detection and Ranging) which they group into pillars instead of voxels allowing them to encode said pillars into a 2D birds eye view pseudo-image of the surroundings.

There are more nuances of Point Pillars which we won't into but its still important that its understood at least at a very basic level how it operates. More information can be found on the Github which can be accessed by clicking the subheader.

Emergency Vehicles

This is the point at which we decided to add a category for emergency vehicles, which is something not considered by the initial project. These include ambulances, fire trucks, and police cars. The first step is simply to work in conjunction with Point Pillars and thus we are looking to initially identify these vehicles and distinguish them from other such automobiles. The next step would to then be able to differentiate if their siren is active or not which would result in different actions needing to be taken, such as pulling over for a cop car or moving to the side to give way for an ambulance.
To identify emergency vehicles we started with an existing starter code doing so on Kaggle using a pretrained model with the Keras and EfficientNet libraries.
For the first part we take the output from the Point Pillars network (the image with cars, pedestrians, and cyclists labeled with the boxes around it) and crop the pictures of cars. We then feed these pictures into the our emergency vehicle detection network and classify the vehicles accordingly. Finally we add the label back onto the original point pillars image so that it can essentially be registered as another category.
